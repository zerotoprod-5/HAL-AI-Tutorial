So what is it, really

Word 4 — Train/test split

Learn from most.
Judge on the hidden rest.

75%

25%

Like an exam: never let the student see the exact paper first. We judge the model only on cases it has never seen.

Session 2 — predict a category

What it got right — and how it's wrong.

  • Predict urgency (a category): ~81% on unseen snags
  • Mistakes mostly Low↔Medium; caught 10/15 High, slipped High→Low once — so High keeps a human's eyes
  • The accuracy trap: always saying "routine" scores high yet catches zero urgent snags

Session 2 — embeddings + clustering

Snags self-group into families — no labels.

  • Each snag becomes a point in space (an embedding); k-means sorts all of them into 5 families
  • A 2-D map shows the families; read each cluster's top words to name it
  • Honest ~0.22 agreement with the true systems — Mechanical pops cleanly, others blend

Session 4 — honest about the limits

A low error rate can still get the one word wrong.

The WER trap

8% — but WRONG part #

WER = wrong + missing + extra words. The "cleaner" 8% transcript still swapped one part number for another. Numbers and serials always get a human's eyes.

Keyword spotting

Listen for 4 commands

A small fixed vocabulary is robust to stutters and filler — why shop-floor tools use wake-words, not full dictation.

Hard by default (accents, jargon, radio noise), strong with domain data — ATC ASR reached >95% controllers, >90% pilots, ~97% on callsigns after domain training.

Neither magic nor nonsense

It was always the same five steps.

  • Text, speech, sound — every session was dataset → split → train → predict → measure
  • The numbers were honest: urgent recall ~95% (one missed), urgency ~81%, downtime ±1.5 h, clustering ~0.22, sound ~0.88, anomaly AUC ~0.94 — and a low WER that still got the one word wrong
  • Neither magic nor nonsense — a tool you can now reason about and question
  • Generative AI — the half that creates — comes in the sessions to follow
